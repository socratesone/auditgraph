This specification outlines the integration of **Gemini 2.0/1.5 Multimodal** capabilities into the [auditgraph](https://github.com/socratesone/auditgraph) architecture. The goal is to evolve the project from a text-based extractor into a high-fidelity, multimodal "Accounting System" for knowledge.

---

# Spec-019: Multimodal Enrichment & Hybrid Retrieval (Gemini Embeddings 2 Integration)

## 1. Objective
Enhance [auditgraph](https://github.com/socratesone/auditgraph) to ingest non-textual data (Video/Audio/Images) and implement a "Hybrid Retrieval" system that combines Gemini’s semantic "intuition" with auditgraph’s [deterministic provenance](https://github.com/socratesone/auditgraph#overview).

## 2. Architectural Changes

### A. The "Multimodal Ingest" Pipeline
Currently, `auditgraph` handles `.md`, `.pdf`, and `.docx`. We will add a `multimodal` stage to the [ingest command](https://github.com/socratesone/auditgraph#cli-reference).

1.  **Stage:** `auditgraph extract --multimodal`
2.  **Logic:** * Files (MP4, MP3, JPG) are sent to Gemini with a system prompt to identify `ag:entity` and `ag:technology` types.
    * **Crucial Requirement:** The LLM must return a **timestamp or frame reference**.
3.  **Output:** A standard `auditgraph` manifest entry where the `source_location` is a timestamp (e.g., `video.mp4#t=01:30`) rather than a line number.

### B. Hybrid Retrieval Engine
Modify the [query behavior](https://github.com/socratesone/auditgraph#query-behavior) to support two-tier searching:

* **Tier 1 (Semantic):** Use `text-embedding-004` (Gemini) to find the "neighborhood" of a query.
* **Tier 2 (Structural):** Once the starting nodes are found via embedding, the system uses [Neo4j Graph Traversal](https://github.com/socratesone/auditgraph#neo4j-export-and-sync) to find deterministic neighbors (references, sections, dependencies).

## 3. New Component Specifications

### 3.1 `auditgraph.extractors.gemini`
A new module to handle the interface with Google's AI.
```python
def extract_multimodal_entities(file_path: str):
    """
    1. Upload file to Gemini File API.
    2. Prompt: "Extract all technologies and key concepts. 
       Return JSON: {entity: string, type: string, timestamp: string}"
    3. Convert to auditgraph 'ag:entity' format.
    4. Log the API Request/Response in the Audit Trail.
    """
```

### 3.2 `auditgraph.linking.inferred`
A new linking logic to complement [content extraction](https://github.com/socratesone/auditgraph#content-extraction).
* **Existing:** `ag:reference` (Hardcoded markdown links).
* **New:** `ag:inferred_link`.
* **Audit Logic:** These links must be visually distinct in the [Neo4j Export](https://github.com/socratesone/auditgraph#neo4j-export-and-sync) (e.g., dashed lines) and must contain a `confidence_score` and `model_version` in the metadata.

## 4. Updated User Workflow

1.  **Initialize with Gemini API Key:**
    ```bash
    export GOOGLE_API_KEY="your_key"
    auditgraph init --multimodal
    ```

2.  **Ingest a Video Tutorial:**
    Place `tutorial.mp4` in the `source/` folder.
    ```bash
    auditgraph ingest --root .
    ```
    *Auditgraph extracts "Docker" and "Kubernetes" entities from the audio/frames and links them to the video file.*

3.  **Hybrid Query:**
    ```bash
    auditgraph query --q "How do I deploy the auth service?" --hybrid
    ```
    *The system uses embeddings to realize you are talking about "Deployment," then uses the Graph to find the specific [ag:section](https://github.com/socratesone/auditgraph#content-extraction) in your documentation linked to "Docker".*

---

## 5. Traceability & Audit Standards
To maintain the project's [value proposition](https://github.com/socratesone/auditgraph#features) of "deterministic tracing":
* **No Black Boxes:** Every "Inferred Link" generated by Gemini must be stored in the `auditgraph` [run manifest](https://github.com/socratesone/auditgraph#overview).
* **Diffability:** Users must be able to run `auditgraph diff` to see exactly which links were added by a model update versus a manual documentation change.
