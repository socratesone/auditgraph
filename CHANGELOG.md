# Changelog

## Unreleased

### Added
- **Spec 028 markdown ingestion produces honest, queryable results.** Bundles five bug fixes from the Orpheus consumer report with the markdown sub-entity extraction capability gap. Highlights:
  - **US1 — Cache-skip fix (BUG-1).** `IngestRecord` gains an orthogonal `source_origin ∈ {fresh, cached}` field. Cache hits now record `parse_status="ok"` + `source_origin="cached"` instead of the pre-028 `parse_status="skipped"` (which was causing extract to silently drop every cached record on rerun — `0 entities from N valid inputs`). A backward-compat reader translates pre-028 manifests in memory. Invariant I6 enforced at `build_source_record`: `failed+cached` and `skipped+cached` are rejected; `parse_status` must be one of `{ok, failed, skipped}`.
  - **US2 — Markdown sub-entity extractor (the capability gap).** New `auditgraph/extract/markdown.py` emits `ag:section` / `ag:technology` / `ag:reference` entities from every markdown source, plus four link rule IDs: `contains_section`, `mentions_technology`, `references`, `resolves_to_document`. Source-scoped deterministic IDs (`ent_<sha256(<source_hash>::<type>::<key>[::<order>])>`); `canonical_key` field is distinct human-readable metadata. Case-fold + whitespace-trim dedup for technology tokens. Fenced code blocks emit one entity keyed on the language info string (NOT body content). Bare-URL detection via `markdown-it-py[linkify]>=4,<5`. Pre-heading content attaches origin edges to the document anchor (note entity). Pruning (FR-016c) removes stale markdown sub-entities before writing refreshed ones on source edit. Images skipped in v1.
  - **US3 — Honest pipeline status (BUG-5).** New `StageManifest.warnings` field, always serialized (even as `[]`). Binary threshold: `no_entities_produced` fires from `run_extract` when upstream has ≥ 1 ok record and extract emits 0 entities; `empty_index` fires from `run_index` when entities ≥ 1 and BM25 postings = 0. Exit code unchanged (warnings, not errors, per FR-019).
  - **US4 — Rule-pack stubs + validator (BUG-2).** Shipped stubs at `auditgraph/_package_data/config/extractors/core.yaml` and `.../link_rules/core.yaml` via `[tool.setuptools.package-data]`. Editable-checkout mirrors at the top-level `config/` directories. `auditgraph init` copies both stubs into new workspaces alongside `pkg.yaml` (idempotent). New `auditgraph/utils/rule_packs.py` validates rule-pack paths at config load time with workspace-root-relative resolution (no more `config/config/...` path doubling), package-resource fallback, and distinct `rule_pack_missing` / `rule_pack_malformed` error kinds. CLI catches `RulePackError` and exits with code 5 + structured JSON envelope.
  - **US5 — `auditgraph node <id>` dispatch fix (BUG-4).** `node_view` rewritten as a table-driven ID-prefix resolver: `doc_*` → `documents/`, `chk_*` → `chunks/`, everything else → `entities/`. Fall-through chain catches pathological cases. Structured not-found envelope replaces raw `FileNotFoundError` for unknown IDs. CLI exits 1 when `node_view` returns an error dict, preserving pre-028 exit-code signaling for scripts.
  - **US6 — Wall-clock timestamps on manifests (BUG-3).** `StageManifest` and `IngestManifest` gain `wall_clock_started_at` / `wall_clock_finished_at` fields populated by the new `wall_clock_now()` helper in `storage/hashing.py`. Existing deterministic `started_at` / `finished_at` preserved — `outputs_hash` stability unchanged across runs. `wall_clock_now` is monkeypatchable for test pinning.
  - **Cooccurrence exclusion (FR-016e).** `build_source_cooccurrence_links` now filters candidate pairs where either endpoint is in `EXCLUDED_COOCCURRENCE_TYPES = {ag:section, ag:technology, ag:reference}` — either-endpoint rule, not just same-type pairs. Prevents O(n²) noise from the new sub-entity classes.
  - **Opt-out flag (FR-016i).** `extraction.markdown.enabled: false` makes BOTH the producer AND the pruning helper inert. Previously-emitted markdown sub-entities are preserved — the flag is a pure activation switch, not a cleanup command.
  - **Source-metadata preservation on cache-hit (regression fix).** Cache-hit rerun no longer clobbers `sources/<hash>.json` with minimal record-only metadata — the fresh-parse's `frontmatter`, `document`, `segments`, `chunks` keys are merged forward. Fixes a duplicate-note-entity bug where the second run's extract fell back from the frontmatter title to the filename stem.
  - **`run_import` parity.** FR-016b1 cache-completeness check applied to the import path too — pre-028 cached markdown documents get refreshed on first post-028 `auditgraph import` instead of raising FR-016b2 at extract time.
  - **`DocumentsIndex` built from current-run records only (adjustments3 §4).** Joins normalized ingest manifest against on-disk document payloads on `source_hash`; no disk scan, no document-id recomputation. Stale document artifacts from prior runs whose sources are absent from the current manifest are correctly excluded — references to their paths classify as `unresolved`.
  - **Migration guidance.** Existing workspaces run through Spec-028 automatically: the first post-028 run reparses markdown files whose cached documents lack the new `text` field (silent one-time migration), emits the three new entity types and four new link types, and prunes any stale markdown sub-entities on subsequent edits. No manual rebuild required. Runs use `parse_status="ok" + source_origin="cached"` going forward. `markdown-it-py[linkify]>=4,<5` is the only new explicit dependency (pulls in `linkify-it-py>=2,<3`).
  - See `specs/028-markdown-extraction/` for the spec, plan, contracts (5), data model (10 invariants I1–I10), research notes (12 threads R1–R12), quickstart (10-step walkthrough), and the 100-task implementation breakdown.

### Security
- **Spec 027 hardening (Phases 2-4 of the security campaign).** Closes both P1 findings from the Spec 026 audit and adds defense-in-depth for the M/H findings flagged for follow-up. Highlights, by user story:
  - **US1 — Symlink containment (FR-001..FR-004a).** `auditgraph ingest` and `auditgraph import` now refuse symlinks whose resolved target escapes the workspace root, with `skip_reason: symlink_refused` recorded in the manifest and a one-line stderr WARN. The reserved `--allow-symlinks` flag exists but raises `NotImplementedError`. New helper `auditgraph/utils/paths.py:contained_symlink_target` is the single source of truth for the check.
  - **US2 — MCP payload validation (FR-005..FR-009).** Every MCP tool-call payload is now validated against its declared `input_schema` via `jsonschema` before any subprocess is invoked. Validation failures return a structured envelope `{error: {code: "validation_failed", tool, field, reason}}` and never echo the rejected value. Server-level default `maxLength=4096` for unconstrained string parameters, configurable via `AUDITGRAPH_MCP_MAX_STRING_LENGTH` (zero/negative/non-numeric values raise `ConfigurationError`). New module `llm-tooling/mcp/validation.py`. Inventory guard test enforces drift-free coverage of every tool in `tool.manifest.json`.
  - **US3 — Export-neo4j path containment (FR-010).** `auditgraph export-neo4j --output <path>` now applies the same `ensure_within_base` check as `auditgraph export`. Absolute external paths are refused. Default output path is `<root>/exports/neo4j/export.cypher`.
  - **US4 — Detector expansion (FR-011..FR-015).** New `cloud_keys` detector category covers AWS (`AKIA...`), Google (`AIza...`), Anthropic (`sk-ant-api...`), OpenAI (`sk-proj-...` and legacy `sk-[A-Za-z0-9]{48}`), Stripe (`sk_live_...`). The `vendor_token` category extends to GitHub fine-grained PAT prefixes (`github_pat_`, `gho_`, `ghu_`, `ghs_`, `ghr_`) and Slack `xoxe.xoxp-`. The `credential_kv` keyword set adds `aws_access_key_id`, `aws_secret_access_key`, `auth_token`, `access_token`, `refresh_token`, `session_token`, `passwd`, `pwd`, `bearer`, `auth`. Per Clarification Q6, `cloud_keys` and `vendor_token` are distinct reporting buckets.
  - **US5 — Cross-chunk PEM containment / M2 fix (FR-016..FR-018).** Redaction is now applied at the parser entry point (`auditgraph/ingest/parsers.py:_build_document_metadata`) **before** chunking, so multi-line secrets like RSA PEM keys that straddle a chunk boundary are caught regardless of chunk size. The Spec 026 C1 hotfix's post-chunking pass has been **retired** in both `run_ingest` and `run_import` — parser-entry is now the single canonical redaction site. `parse_file` now requires `parse_options["redactor"]` and raises `ValueError` if missing (silent no-op was the exact failure mode C1 closed). New `tests/support.py:null_parse_options()` helper builds an explicit empty-detector redactor for tests that don't care about redaction.
  - **US6 — `auditgraph validate-store` command (FR-019..FR-022).** New read-only command audits an existing `.pkg/profiles/<profile>/` store for strings matching the current detector allowlist, without requiring a full rebuild. Supports `--profile`, `--all-profiles`, and `--format json|text`. Scope (per Clarification Q5) is `entities/`, `chunks/`, `segments/`, `documents/`, `sources/` only — explicitly excludes `runs/`, `indexes/`, `secrets/`. Read-only contract verified by mtime preservation tests. Exposed as MCP tool `ag_validate_store`. Shard walking and detection are delegated to the shared helper `auditgraph/query/_shard_scanner.py:scan_shards_for_misses` per Constitution Principle I (DRY).
  - **US7 — Neo4j plaintext URI warning (FR-023, FR-023a).** `load_connection_from_env` now parses the URI host. Non-loopback `bolt://`/`neo4j://` URIs emit a one-line stderr WARN. With the new `--require-tls` flag on `sync-neo4j` (or `AUDITGRAPH_REQUIRE_TLS=1` in the environment), the warning is escalated to `Neo4jTlsRequiredError` and the CLI exits with code 4. Loopback literals (`localhost`, `127.0.0.1`, `::1`) are exempt regardless of strict mode and regardless of port.
  - **US8 — Pipeline redaction postcondition (FR-024..FR-028).** After `auditgraph rebuild` finishes the `index` stage, a postcondition walks the canonical shard directories and re-runs the detector set. On match, the rebuild raises `PostconditionFailed` and the CLI exits with code 3 unless `--allow-redaction-misses` was passed (then `status: "tolerated"`, exit 0). The `redaction_postcondition` block is merged into `index-manifest.json` per `contracts/postcondition-manifest.md`. Prior-stage failures emit `status: "skipped"`. Pipeline redaction is now self-enforcing.
  - **Migration guidance for users with pre-Phase-1 stores.** Run `auditgraph validate-store` against your existing `.pkg/`. If it reports any misses, run `auditgraph rebuild` — the new parser-entry redaction will scrub any cleartext that the post-chunking pass missed, and the postcondition will gate the new run. For multi-profile stores, use `--all-profiles`. The shared helper, the new `cloud_keys` category, the symlink-refusal records, and the postcondition manifest field are all additive — no schema migration is required.
  - **Dependencies.** `jsonschema>=4,<5` added (~200 KB pure-Python, used by MCP validation). Lower-bound pins added for `pyyaml>=6.0.3`, `pypdf>=6.9.1`, `python-docx>=1.2.0` (no upper bounds; full dependency-pinning policy is deferred to a future spec per Clarification Q8).
  - See `specs/027-security-hardening/` for the spec, plan, contracts, and the 124-task implementation breakdown.

- **CRITICAL: redactor was bypassed for `chunks/`, `segments/`, and `documents/` shards (Spec 026 C1).** Previously only `.pkg/profiles/<profile>/sources/<hash>.json` was routed through the redactor — document bodies carrying credential-shaped strings persisted in **cleartext** under `chunks/`, `segments/`, and `documents/`. `auditgraph import <path>` was strictly worse: it wrote *every* artifact (including `sources/`) with plain `write_json`, bypassing redaction entirely. Any backup tool, cloud sync client, or accidental `git add .pkg` would exfiltrate secrets the user thought the redactor had scrubbed. The fix (`auditgraph/pipeline/runner.py`) routes document/segment/chunk payloads through `redactor.redact_payload(...)` before `write_document_artifacts` in both `run_ingest` and `run_import`, and restores `write_json_redacted` for the source artifact in `run_import`. New regression tests in `tests/test_spec011_ingest_redaction.py` walk every shard directory after ingest *and* import and assert the sentinel is absent — both tests were verified to fail against the unfixed code and pass after the fix. **If you have an existing `.pkg/` store that ingested sensitive material before this release, re-run `auditgraph rebuild` and consider whether any backups or sync destinations may hold the poisoned artifacts.** See `specs/026-security-hardening/NOTES.md` for the full tri-scanner audit writeup and the remaining H/M/L findings that will ship as Spec 026 Phase 2.

### Removed
- **Source code symbol extraction is permanently out of scope (Spec 025).** Auditgraph is a documents + provenance tool. Files with `.py`, `.js`, `.ts`, `.tsx`, `.jsx` extensions are no longer ingested by default — they are skipped at the ingest stage with reason `unsupported_extension`. Specifically:
  - Deleted `auditgraph/extract/code_symbols.py` entirely. The function was named `extract_code_symbols` but only ever produced one opaque `file` entity per source file with no real symbol information — the name was a lie about what it did.
  - Deleted the `text/code` parser_id from `auditgraph/ingest/policy.py:PARSER_BY_SUFFIX` and the corresponding entries from `DEFAULT_ALLOWED_EXTENSIONS`.
  - Deleted the `chunk_code.enabled` opt-in config flag (added briefly during the post-Spec-023 quality sweep) and its supporting wiring across `auditgraph/pipeline/runner.py`, `auditgraph/ingest/parsers.py`, `auditgraph/config.py`, and `config/pkg.yaml`.
  - Deleted `tests/test_code_chunking_opt_in.py`.
  - Deleted the now-dead `build_entity` function from `auditgraph/extract/entities.py` (its only call site was inside the deleted code symbols loop).
  - For code structure navigation, use a language-aware tool: LSP, ctags, ripgrep, treesitter-based analyzers, `tldr`, your IDE, or any of the many existing code intelligence products.
  - **Migration**: existing workspaces with file entities for `.py` etc. will see those entities re-created with identical IDs by git provenance (Spec 025 fix below) for any path that exists in git history. File entities for code files NOT in git history will not be re-created — to query the new state, run `auditgraph rebuild` after upgrading.

### Fixed
- **Pre-existing dangling-reference bug in git provenance (Spec 025).** Before this change, git provenance emitted `modifies` links pointing at file entity IDs computed via `entity_id(f"file:{path}")`, but the target entities were only ever created by `extract_code_symbols` and only for the 5 code extensions. For every non-code file in any commit's history (markdown, YAML, README, PDF, configs, etc.), the `modifies` link was a dangling reference. After this change, file entity creation is moved into `auditgraph/git/materializer.py:build_file_nodes` and produces an entity for **every** distinct path in commit history, regardless of file extension. `auditgraph neighbors <commit_id> --edge-type modifies` now returns resolvable file entities for ALL modified files. The IDs match what existing `modifies` links point at, so existing workspaces continue to work without migration.

### Added
- **Local query filters & aggregation (Spec 023).** You can now browse, filter, sort, and count entities directly from the CLI without writing post-processing scripts.
  - New `auditgraph list` command — browse entities without a keyword search. Supports `--type`, `--where "field<op>value"`, `--sort`, `--desc`, `--limit`, `--offset`, `--count`, and `--group-by`.
  - Extended `auditgraph query` with the same filter/sort/limit/aggregation flags so you can narrow BM25 search results.
  - Extended `auditgraph neighbors` with `--edge-type` and `--min-confidence` to filter graph traversals by relationship type and confidence threshold.
  - Filter operators: `=`, `!=`, `>`, `>=`, `<`, `<=`, `~` (substring contains). Numeric values are auto-detected; array fields support membership semantics for `=`/`!=` and substring search for `~`.
  - New MCP tools: `ag_list` (with `total_count`/`truncated` response envelope and a default `limit=100`), and the `ag_query` and `ag_neighbors` tools now accept the new filter parameters.
  - New per-type indexes built during the `index` pipeline stage: `indexes/types/<type>.json` and `indexes/link-types/<type>.json` enable selective entity loading without scanning the full corpus.
  - The forward adjacency index (`indexes/graph/adjacency.json`) is now rebuilt from **all** link files, including git-provenance links (commits, authors, modifies, parent_of) — these were previously missing from graph traversals.
  - 161 new tests covering the index builders, filter engine, sort/pagination, aggregation, edge cases, and MCP tool surface.
- Neo4j export/sync feature scaffolding and CLI commands (`export-neo4j`, `sync-neo4j`).
- Neo4j records/cypher builder/export/sync modules under `auditgraph/neo4j/`.
- Targeted Neo4j test suite (`tests/test_neo4j_*.py`) and fixtures.
- Documentation for Neo4j setup and usage in README, QUICKSTART, MCP guide, and environment setup docs.
