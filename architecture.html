auditgraph Architecture

Local-first, deterministic personal knowledge graph CLI · sharded JSON storage · optional Neo4j export

Deterministic Pipeline

  • • 6 stages: ingest → git-provenance → normalize → extract → link → index
  • • inputs_hash/outputs_hash per stage manifest
  • • Deterministic run_id and timestamps
  • • Sort with stable tiebreakers (typically entity.id)
  • • replay-log.jsonl captures every write
  • • No network calls in core path — local-first by design

Security Hardening (Spec 027)

  • • Redactor required at parse_file entry — no silent fallback
  • • Post-index postcondition scan → exit 3 on credential-shape hits
  • • --allow-redaction-misses tolerated-mode escape hatch
  • • Symlink containment at ingest (US1)
  • • AUDITGRAPH_REQUIRE_TLS=1 gates Neo4j export (exit 4)
  • • Pinned deps: pyyaml, pypdf, python-docx, jsonschema<5
  • • Shared scan_shards_for_misses used by validate-store & postcondition

Storage — Sharded JSON

  • • Root: .pkg/profiles/<profile>/
  • • One file per entity/link/chunk: <shard>/<id>.json
  • • 2-char shard derived via storage.sharding.shard_dir
  • • All I/O through read_json/write_json (sort_keys, indent)
  • • Indexes rebuilt from materialized entities + all link files
  • • Source code extensions skipped at ingest (Spec 025, out of scope)

MCP Tooling

  • • tool.manifest.json is the single source of truth
  • • skill.md + OpenAI adapters are generated
  • • Read tools: timeout ≤ 5000 ms; every tool has ≥1 example
  • • Contract tests in llm-tooling/tests/ enforce manifest shape
  • • 14 read + 10 write tools covering the full CLI surface
  • • Adding a new tool: manifest + mcp_inventory.py + payload-validation guard

Query Surface

  • • query (BM25), list, neighbors, why-connected, node, diff
  • • Git-aware: git-log, git-who, git-introduced, git-history
  • • Local filters (Spec 023): --type --where --sort --limit --count --group-by
  • • Edge filters: --edge-type --min-confidence
  • • Loaders return generators — deterministic iteration order
  • • All CLI output through cli._emit() (no stray prints)

Optional / Opt-In

  • • NER via spaCy — enabled: false by default
  • • Natural-language extension allowlist gates NER per chunk
  • • Neo4j export via sync-neo4j — offline target only
  • • LLM provider interface present (NullProvider only shipped)
  • • Jobs runner (jobs.yaml) for automating stage chains
  • • No backwards-incompatible CLI changes — new flags are additive
